## 0. 기술 스택 / 공통 원칙 (A와 동일)

- React(Vite)+TypeScript, Tailwind CSS, **A의 디자인 토큰 상속**
- 정적 배포, 백엔드 없음. 폼은 외부 서비스(Formspree 등) 또는 mailto
- **localStorage/sessionStorage/DB 사용 금지**
- 반응형 필수: 모바일(≤480)/태블릿(≤768)/데스크톱(≥1024)
- 라우트: `/mosaic-guard` (A의 라우터에 편입)

### ★공통 처리 규칙 — 실증 숫자 placeholder
```
- 모든 미확정 숫자/지표는 placeholder 변수로 분리 (예: const METRICS = { reviewTimeReduction: "[00]", ... })
- 화면 표기: 회색 + 【TODO: 실측값 입력】 배지
- 한 파일(예: src/data/metrics.ts)에 모아 나중에 일괄 교체
- 【공개/제출 전 체크리스트】에 "TODO 배지 잔존 여부 확인" 포함
```

### ★공통 처리 규칙 — 비주얼 역할 분담
```
- 힉스필드(AI 생성 이미지/영상): Hero 배경, 증적 섹션의 '분위기' 추상 비주얼만
- 코드(SVG/CSS): 핵심 논리 다이어그램(집적위험 격상, 결정론 vs 확률, 파이프라인) — 정밀·정확이 중요하므로 코드로 직접 작도
- 모든 비주얼: 군사 이미지·실제 부대·무기·좌표 절대 금지. 추상화 필수.
- 영상 사용 시 모바일은 정적 이미지 fallback (로딩 성능)
```

### 블록 2 — 문제 정의 (★페이지의 심장)
**메시지**: 왜 결합이 위험한가 — SnapScale의 "왜 설계변경이 위협인가" 대응

- H2: "왜 *안전한 문서들*이, 합쳐지면 위험해지는가"
- **핵심 다이어그램 (코드/SVG, 반응형) ★최우선 비주얼**:
  ```
  [문서 A: 부대명]  [문서 B: 위치]  [문서 C: 시간]  [문서 D: 규모]
        (각각 '안전' 녹색)
              │  AI 결합·요약  │
              ▼
     [산출물: 부대 이동 정보]  ← '위험' 빨강으로 격상
  ```
  - 데스크톱: 가로 4열 → 화살표 → 결과. 모바일: 세로 스택.
  - 스크롤 인터랙션: 개별 문서(녹색)가 차례로 등장 → 결합 시 결과가 빨강으로 전환되는 모션. **이 한 컷이 제품 전체를 설명.**
  - ※ 모든 예시는 **가상 더미 데이터**. 화면에 "예시는 가상 데이터입니다" 캡션 명시.
- 보조 통계 3종(SnapScale식, placeholder):
  | 값 | 라벨 |
  |---|---|
  | `[00]` | 기존 보안이 놓치는 결합 위험 비율 【TODO/출처필요】 |
  | `증가` | AI 도입 후 검수 대상 문서량 |
  | `결재자` | 미검수 산출물 사고 시 책임 귀속 |

### 블록 3 — 왜 규칙 기반인가 (WHY RULE-BASED)
**메시지**: SnapScale "왜 파인튜닝 안 하나" 대응 — 우리의 기술 선택 방어 논리

- H2: "왜 *AI 추론*이 아니라 *규칙*으로 판정하는가"
- 부제: "면책 증적에는 '재현 가능성'이 필수입니다. 그래서 우리는 결정론적 규칙을 택했습니다."
- **3개 장벽 카드** (SnapScale BARRIER 형식):
  | # | 제목 | 내용 |
  |---|---|---|
  | 01 | 재현성 | 확률적 AI 판정은 같은 입력에도 흔들려 사후 감사 불가 |
  | 02 | 설명가능성 | "왜 위험한가"를 규칙 ID로 추적·제시 가능 |
  | 03 | 책임소명 | 결정론적 판정만이 '적법 절차 이행'을 입증 |
- **결론 박스**(SnapScale식): "AI가 판정하지 않습니다. *규칙이 환기하고, 사람이 결정합니다.*"
- 비주얼: 결정론 vs 확률 비교 도식(코드/SVG). 같은 입력 → 규칙은 동일결과 / AI는 산포.

### 블록 4 — 솔루션 / 작동 (HOW IT WORKS)
**메시지**: 4단계 파이프라인 + 작동 데모

- H2: "MOSAIC-Guard는 이렇게 작동합니다"
- **4단계 흐름** (코드 다이어그램, 반응형 — 데스크톱 가로/모바일 세로):
  `① 요소 추출·분류 → ② 결정론적 집적위험 판정 → ③ 출처 역추적 + 출구 제시 → ④ 해시체인 증적·인증서`
  - 각 단계 클릭/호버 시 1줄 설명 펼침
- **작동 데모 영역**:
  - UI 목업 스크린샷(좌: 위험 문장 / 우: 원문 하이라이트 매핑) — **더미 데이터, "가상 데이터" 명시**
  - 가능하면 짧은 데모 영상/GIF (실제 엔진 데모 완성 후 교체) 【TODO: 데모 영상 자리】
- 비주얼: 실제 제품 UI 느낌의 목업. 정적 이미지 또는 간단한 인터랙티브 위젯.

### 블록 5 — 차별점 (vs 기존)
**메시지**: 경쟁 비교표

- H2: "기존 보안 솔루션과 무엇이 다른가"
- **비교표** (반응형 — 모바일에선 카드 전환):
  | 구분 | 전통 보안/DLP | LLM 가드레일 | **MOSAIC-Guard** |
  |---|---|---|---|
  | 통제 대상 | 패턴/키워드 | 입력 프롬프트 공격 | **결합 집적위험·절차 무결성** |
  | 모자이크 효과 | 대응 불가 | 제한적 | **정면 대응** |
  | 책임 소명 | 차단 로그 | 차단 알람 | **해시체인 증적·인증서** |
- 강조: MOSAIC-Guard 열을 accent로 하이라이트.
