## 0. 기술 스택 / 공통 원칙 (A와 동일)

- React(Vite)+TypeScript, Tailwind CSS, **A의 디자인 토큰 상속**
- 폰트 **Pretendard** (fallback Noto Sans KR)
- 정적 배포, 백엔드 없음. 폼은 외부 서비스(Formspree 등) 또는 mailto
- **localStorage/sessionStorage/DB 금지**
- 반응형 필수: 모바일(≤480)/태블릿(≤768)/데스크톱(≥1024)
- 라우트: `/mosaic-guard` (A의 라우터에 편입)
- **다크 단일 모드**로 구현(정량·표 슬라이드만 라이트 반전 허용)

### ★공통 처리 규칙 — 실증 숫자 placeholder
```
- 모든 미확정 숫자/지표는 placeholder 변수로 분리 (예: src/data/metrics.ts → { reviewTimeReduction: "[00]", ... })
- 화면 표기: 회색 + 【TODO: 실측값 입력】 배지
- 한 파일에 모아 나중에 일괄 교체
- 【공개/제출 전 체크리스트】에 "TODO 배지 잔존 여부 확인" 포함
```

### ★공통 처리 규칙 — 비주얼 역할 분담
```
- 힉스필드(AI 생성 이미지/영상): Hero 배경, 증적 섹션의 '분위기' 추상 비주얼만
- 코드(SVG/CSS): 핵심 논리 다이어그램(집적위험 격상, 결정론 vs 확률, 파이프라인) — 정밀·정확이 중요하므로 코드로 직접 작도
- 모든 비주얼: 군사 이미지·실제 부대·무기·좌표 절대 금지. 추상화 필수.
- 영상 사용 시 모바일은 정적 이미지 fallback (로딩 성능)
```

## 1. 디자인 시스템 (★A와 동일 토큰 — 상속)

| 토큰 | 값 | 용도 |
|---|---|---|
| `bg-base` | `#000905` | 배경 (딥블랙그린) |
| `bg-surface` | `#0C1712` | 카드·표면 |
| `bg-elevated` | `#13201A` | 호버·강조 |
| `text-primary` | `#D8E8DB` | 본문 |
| `text-muted` | `#8BBB94` | 보조 |
| `accent` | `#8BBB94` / `#A9D4B4` | CTA·포인트 |
| `brand-deep` | `#004225` | 로고·강조 블록 |
| `border-subtle` | `#1A2620` | 경계선 |
| **`danger`** | **`#D90024`** | **위험 격상 전용 (★B 핵심 다이어그램에 사용)** |
| **`safe`** | **`#052E5C` (다크 위 `#1A4D7A` 보정)** | **안전 상태 (★격상 전 '안전 문서')** |
| `surface-light` | 배경 `#FEFEFA` / 텍스트 `#004225` | 라이트 영역(표·정량) |

> ★ B에서 danger·safe가 실제로 핵심 역할: "안전 문서(safe/녹색) → 결합 → 위험 격상(danger/레드)" 다이어그램의 색이 곧 메시지.
> danger는 **"격상" 표현에만**. 제목·테두리·일반 강조에 쓰지 말 것(남발 시 톤 붕괴).

## 3. 페이지 구조 (SnapScale 골격 차용, 9블록)

순서: **Hero → 문제 → 왜 규칙인가 → 솔루션(작동) → 차별점 → 증적/면책 → 실증/VoC → 폼 → 푸터**

### 블록 1 — Hero
- 상단 뱃지: `국방창업경진대회 진행 중` · `특허 출원 완료` (← "등록" 아님)
- 카테고리 라벨: `ON-PREMISE · RULE-BASED · TAMPER-EVIDENT`
- **H1**: "개별로는 안전한 정보, 합쳐지면 기밀이 됩니다."
- 부제: "AI가 만든 문서의 집적 위험을 탐지하고, 검수 절차를 위변조 없이 증적화합니다. 판정은 규칙이, 결재는 사람이."
- CTA 2개: 주 `데모/도입 문의 →`(폼으로) / 부 `어떻게 작동하나 →`(솔루션으로)
- **Hero 통계 4종** (카운트업, placeholder):
  | 값 | 라벨 |
  |---|---|
  | `[00]%` | 검수 시간 절감 【TODO】 |
  | `[00]초` | 문서 N건 검수 처리 시간 【TODO】 |
  | `100%` | 완전 온프레미스(외부 연동 0) |
  | `[00]+` | 집적위험 룰셋 규칙 수 【TODO】 |
- 배경: 위 2번 영상 규칙

### 블록 2 — 문제 정의 (★페이지의 심장)
- H2: "왜 *안전한 문서들*이, 합쳐지면 위험해지는가"
- **핵심 다이어그램 (코드/SVG, 반응형) ★최우선 비주얼**:
  ```
  [문서 A: 부대명] [문서 B: 위치] [문서 C: 시간] [문서 D: 규모]
       (각각 safe色 #8BBB94 '안전')
              │  AI 결합·요약  │
              ▼
     [산출물: 부대 이동 정보]  ← danger色 #D90024 '위험' 격상
  ```
  - 데스크톱 가로 4열→화살표→결과 / 모바일 세로 스택
  - 스크롤 인터랙션: 안전 문서(녹색) 차례 등장 → 결합 시 결과가 **#D90024로 전환**되는 모션
  - ★ 녹색(safe)→레드(danger) 색전환이 시각적 클라이맥스. 이 한 컷이 제품 전체를 설명
  - ※ 모든 예시 = **가상 더미 데이터**, "예시는 가상 데이터입니다" 캡션 명시
- 보조 통계 3종(placeholder): 결합위험 비율【TODO/출처】 / 문서량 증가 / 결재자 책임귀속

### 블록 3 — 왜 규칙 기반인가 (WHY RULE-BASED)
- H2: "왜 *AI 추론*이 아니라 *규칙*으로 판정하는가"
- 부제: "면책 증적에는 '재현 가능성'이 필수입니다. 그래서 결정론적 규칙을 택했습니다."
- **3개 장벽 카드** (SnapScale BARRIER 형식):
  | # | 제목 | 내용 |
  |---|---|---|
  | 01 | 재현성 | 확률적 AI 판정은 같은 입력에도 흔들려 사후 감사 불가 |
  | 02 | 설명가능성 | "왜 위험한가"를 규칙 ID로 추적·제시 |
  | 03 | 책임소명 | 결정론적 판정만이 '적법 절차 이행'을 입증 |
- 결론 박스: "AI가 판정하지 않습니다. *규칙이 환기하고, 사람이 결정합니다.*"
- 비주얼: 결정론 vs 확률 비교 도식(코드/SVG) — 같은 입력 → 규칙은 동일결과 / AI는 산포
