# AllDayProject 팀 랜딩(A) 개발 사양서
## — Claude Code 전달용 —

> 이 문서는 **모브랜드 팀 랜딩페이지(A)** 의 개발 사양입니다.
> Codespace + Claude Code 환경에서 이 사양을 기준으로 구현합니다.
> 제품 랜딩(B, MOSAIC-Guard)은 별도 사양서로 이어집니다. **A에서 정의한 디자인 토큰을 B가 상속**하므로, 토큰 정의를 재사용 가능한 형태로 분리해 구현할 것.

---

## 0. 기술 스택 / 구조

- **프레임워크**: React (Vite) + TypeScript — 정적 배포 전제
- **스타일**: Tailwind CSS (디자인 토큰을 `tailwind.config`에 정의 → B와 공유)
- **배포 가정**: 정적 호스팅(Vercel/Netlify/GitHub Pages), 백엔드 없음
- **라우팅 구조** (엄브렐러/서브디렉터리):
  ```
  /                → 팀 랜딩 (본 사양, A)
  /mosaic-guard    → 제품 랜딩 (B, 추후)
  /millog          → 포트폴리오 (추후)
  ```
  → React Router로 경로 분기. A는 `/` 라우트.
- **폼 처리**: 서버리스. 외부 폼 서비스(Formspree/Getform 등) 또는 mailto. **localStorage/DB 사용 안 함.**
- **반응형 필수**: 모바일(≤480) / 태블릿(≤768) / 데스크톱(≥1024) 3 브레이크포인트.

## 1. 디자인 시스템 (★B가 상속 — 토큰으로 분리 정의)

### 컬러 (다크 뉴트럴)
| 토큰 | 값(제안) | 용도 |
|---|---|---|
| `bg-base` | `#0B0D10` (거의 검정) | 페이지 배경 |
| `bg-surface` | `#14181D` (차콜) | 카드·섹션 표면 |
| `bg-elevated` | `#1C2128` | 호버·강조 표면 |
| `text-primary` | `#F2F4F7` | 본문 |
| `text-muted` | `#9AA4B2` | 보조 텍스트 |
| `border-subtle` | `#262C34` | 경계선 |
| **`accent`** | **`#3B82F6` (블루) 또는 `#10B981` (실버그린) 중 택1** | 포인트색 ★1개만 |

> accent 1개 원칙. 보안·신뢰 도메인엔 차분한 블루(#3B82F6) 권장. 실버그린(#10B981)은 "통과/안전" 뉘앙스가 강함. **택1 후 전 페이지 동일 적용.**

### 타이포그래피
- 한글: **Pretendard** (웹폰트, 보안/테크 톤에 최적) — CDN 또는 self-host
- 영문/숫자: Pretendard로 통일하거나, 숫자 강조엔 `Inter`/`Space Grotesk`
- 스케일: Hero 48~64 / H2 28~32 / H3 20 / body 16 / caption 13
- Hero 통계 숫자는 **크고 굵게**(폰트 700~800), SnapScale형

### 톤·무드
- 차분하고 절제됨. 과한 그라데이션·네온 지양.
- 여백 충분히. 다크 배경에 정보 밀도는 낮게(현관답게).
- 마이크로 인터랙션: 스크롤 시 페이드인, 숫자 카운트업 정도만. 과하지 않게.

## 2. 페이지 구조 (5블록, 1스크롤)

### 블록 1 — Hero (★가장 중요)
**목적**: 3초 각인 + B로 송출

- **상단 내비**: 좌측 `AllDayProject` 로고(텍스트) / 우측 메뉴 `Products · Team · Contact` (앵커 스크롤). 우측 끝 `MOSAIC-Guard →` 버튼(accent).
- **태그라인(메인 카피)**:
  - 메인: **"언제나 도전한다"** 를 직역 슬로건으로 쓰지 말 것. 대신 실행력을 전면에:
    - H1(예시): **"빠르게 만들고, 빠르게 증명한다"**
    - 부제: "현장의 문제를 기술로 푸는 팀, AllDayProject"
  - → 슬로건("언제나 도전")은 부제나 about에 보조로 배치.
- **Hero 통계 4종** (카운트업 애니메이션):
  | 숫자 | 라벨 | 비고 |
  |---|---|---|
  | `6주` | Millog 웹·안드로이드 베타 출시 + 사용자 인터뷰 | **[검증 필요: 정확한 기간]** |
  | `2주` | MOSAIC-Guard 특허 출원번호 확보 | **[검증 필요]** / "출원"으로 정직 표기 |
  | `2개` | 출시·개발 중 프로덕트 | |
  | `N+` | (수상/인터뷰 수 등 — 채울 숫자 확정 필요) | **[값 미정]** |
- **CTA 버튼 2개**:
  - 주(accent, 채움): **`MOSAIC-Guard 보기 →`** → `/mosaic-guard`
  - 부(외곽선): **`팀 소개`** → `#team` 앵커 스크롤
- **배경**: 힉스필드로 생성할 추상 비주얼/영상 자리(placeholder). 어두운 추상 그래픽. **군사 이미지·실제 장비·부대 금지.**

### 블록 2 — Mission (What we do)
**목적**: 팀 미션 한 문장

- H2: 짧게. (예시) **"AI가 만든 결과를, 사람이 안심하고 책임질 수 있게."**
- 보조 1~2줄: 팀이 푸는 문제의 결을 설명. 여기서 "빠른 추진력·실행력"을 한 번 더 언급.
- 레이아웃: 중앙 정렬, 텍스트만. 미니멀.

### 블록 3 — Products (★A의 관문)
**목적**: B로 보내는 실질 동선

- H2: **"우리가 만든 것들"** 또는 **"Products"**
- 카드 2개, **시각적 비중 차등**:
  - **카드 1 — MOSAIC-Guard (大, 메인)**:
    - 태그: `진행 중 · 국방창업경진대회` / `특허 출원 완료`
    - 한 줄: "AI 산출물의 집적위험을 탐지하고 검수 절차를 증적화하는 폐쇄망 보안 엔진"
    - CTA: `자세히 보기 →` → `/mosaic-guard`
    - 카드 크게, accent 테두리/호버 강조
  - **카드 2 — Millog (小, 보조)**:
    - 태그: `베타 출시 · 사용자 인터뷰 완료`
    - 한 줄: **[Millog 한 줄 설명 — 사용자 직접 작성]**
    - CTA: `보기 →` → `/millog` (또는 외부 링크)
    - 카드 작게
- **확장성 메모(주석)**: 미래 프로덕트는 카드만 추가하면 되는 그리드 구조로 구현(엄브렐러 전략).

### 블록 4 — Team
**목적**: 신뢰 + 연속성 (사진 없음, 실명+역할+소속)

- H2: **"Team"** / 부제: 성장 서사 한 줄
  - (예시) **"Millog에서 MOSAIC-Guard로 — 한 번 하고 마는 팀이 아닙니다."**
- 팀원 카드 4인 (사진 없이, 이니셜 아바타 또는 역할 아이콘):
  | 이름 | 역할 | 소속 |
  |---|---|---|
  | 진덕명 | CEO | 한국기술교육대 |
  | 이재민 | CPO | 포항공과대 |
  | 김성민 | CMO | 아주대 |
  | 신현규 | CDO | 경희대 |
  > ※ 역할 표기는 사업계획서 기준. 변경 시 반영.
  > ※ 전원 현역 복무 중 — 사진·연락처 등 개인 식별정보 미노출.
- (선택) 자문: 윤상혁 교수(동국대) 한 줄 — 넣을지 사용자 확인.

### 블록 5 — Contact / Footer
**목적**: 연결 (가벼운 폼)

- H2: **"함께하기"** 또는 **"Contact"**
- 가벼운 폼: 이름 / 이메일 / 메시지 + 전송(외부 폼 서비스). **무겁지 않게.**
  - 진짜 리드 수집은 B에서 하므로 A는 최소.
- 또는 폼 대신 `팀 이메일` + `MOSAIC-Guard 문의 →`(B로) 만으로도 가능.
- 푸터: `© 2026 AllDayProject` / 간단한 메뉴 / (보유 시 SNS).

## 3. 보안·정직성 체크리스트 (구현 시 필수 준수)

- [ ] 군사 이미지·실제 부대·무기·좌표 **일절 없음** (Hero 비주얼 포함)
- [ ] 특허는 **"출원/가출원 완료"** 로만 표기. "등록/보유" 금지
- [ ] Hero 숫자(6주/2주 등)는 **사실과 정확히 일치** — 제출 전 검증
- [ ] MOSAIC-Guard 내부 메커니즘(룰셋·가중치) **노출 안 함** — A는 한 줄 설명까지만
- [ ] 군 관련 기관 로고·레퍼런스 **암시 금지** (없는 신뢰 만들지 말 것)
- [ ] 팀원 개인 식별정보(사진·연락처·부대) 미노출

## 4. 구현 우선순위 (Claude Code 작업 순서 제안)

1. 프로젝트 셋업(Vite+React+TS+Tailwind) + 디자인 토큰 정의 (`tailwind.config`)
2. 공통 레이아웃/내비/푸터 컴포넌트 (B와 공유 전제로 분리)
3. 블록 1(Hero) — 통계 카운트업 포함
4. 블록 2~5 순차
5. 반응형 점검(3 브레이크포인트)
6. 힉스필드 비주얼 자리 교체
7. 폼 외부 서비스 연결
8. `/mosaic-guard`, `/millog` 라우트는 빈 페이지로 스텁만 만들어 링크 연결
