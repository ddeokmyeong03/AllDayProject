# AllDayProject 팀 랜딩(A) 개발 사양서 [v2]
## — Claude Code 전달용 —

> **v2 변경점**: 디자인 시스템을 녹색 팔레트로 확정 / 다크모드 메인 / Hero 배경 영상 실제 적용 / 토큰을 B·PPT와 통일.
> 경로: `alldayproject.com/` (모브랜드 팀 랜딩)
> **A에서 정의한 디자인 토큰을 B가 상속**하므로 토큰을 재사용 가능하게 분리 구현.

---

## 0. 기술 스택 / 구조

- **프레임워크**: React (Vite) + TypeScript — 정적 배포
- **스타일**: Tailwind CSS (디자인 토큰을 `tailwind.config`에 정의 → B·PPT와 공유)
- **폰트**: **Pretendard** (fallback: Noto Sans KR) — CDN 또는 self-host
- **배포 가정**: 정적 호스팅(Vercel/Netlify/GitHub Pages), 백엔드 없음
- **라우팅 구조** (엄브렐러/서브디렉터리):
  ```
  /                → 팀 랜딩 (본 사양, A)
  /mosaic-guard    → 제품 랜딩 (B, 추후)
  /millog          → 포트폴리오 (추후, 스텁만)
  ```
- **폼 처리**: 서버리스. 외부 폼 서비스(Formspree 등) 또는 mailto. **localStorage/DB 금지.**
- **반응형 필수**: 모바일(≤480) / 태블릿(≤768) / 데스크톱(≥1024)
- **모드**: **다크 단일 모드로 구현** (라이트는 추후. 첫 버전은 다크만 완성도 우선)

## 1. 디자인 시스템 (★확정 — A·B·PPT 공통 토큰)

### 컬러 (녹색 팔레트 / 다크 메인)
| 토큰 | 값 | 용도 |
|---|---|---|
| `bg-base` | `#000905` | 페이지 배경 (딥블랙그린, 순검정 아님) |
| `bg-surface` | `#0C1712` | 카드·섹션 표면 (배경보다 살짝 밝게) |
| `bg-elevated` | `#13201A` | 호버·강조 표면 |
| `text-primary` | `#D8E8DB` | 본문 |
| `text-muted` | `#8BBB94` | 보조 텍스트 |
| `accent` | `#8BBB94` (강조 시 `#A9D4B4`) | CTA·포인트 |
| `brand-deep` | `#004225` | 로고·강조 블록 |
| `border-subtle` | `#1A2620` | 경계선 |
| `danger` | `#D90024` | 위험 표시 전용(팀 랜딩에선 거의 미사용) |
| `safe` | `#052E5C` (다크 위 `#1A4D7A`로 보정) | 안전 상태 |
| `surface-light` | 배경 `#FEFEFA` / 텍스트 `#004225` | 라이트 영역(추후 반전용) |

> accent는 녹색 1색 계열. 한 화면에 강조는 1~2곳만.

### 타이포그래피 (Pretendard)
- 한글·영문·숫자 모두 Pretendard로 통일
- 스케일: Hero 48~64 / H2 28~32 / H3 20 / body 16 / caption 13
- Hero 통계 숫자는 크고 굵게(700~800)
- **숫자 강조가 핵심** — 실행력 숫자가 이 페이지의 무기

### 톤·무드
- 차분·절제·미니멀(현관). 과한 그라데이션/네온 금지
- 다크 배경에 정보 밀도 낮게. 여백 충분히
- 마이크로 인터랙션: 스크롤 페이드인 + 숫자 카운트업 정도만

## 2. ★Hero 배경 영상 적용 규칙 (신규 — 실제 영상 생성 완료)

힉스필드로 생성한 **추상 모션 배경 영상**(딥블랙그린 배경에 세이지그린 입자가 드리프트하며 네트워크로 연결되는 4초 무음 루프)을 Hero 배경으로 사용.

### 구현 규칙
```html
<!-- Hero 배경 영상 -->
<video
  autoplay muted loop playsinline
  poster="/hero-poster.jpg"   <!-- 1번 정지 이미지 = 로딩 전/모바일 fallback -->
  class="hero-bg-video">
  <source src="/hero-bg.mp4" type="video/mp4" />
</video>
```
- **속성 필수**: `autoplay muted loop playsinline` (muted 없으면 자동재생 차단됨)
- **모바일 fallback**: ≤768px에서는 영상 대신 `poster` 정지 이미지만 표시
  (영상이 모바일 데이터·배터리 소모 → CSS media query로 `<video>` 숨기고 정지 이미지 배경)
- **오버레이**: 영상 위에 어두운 반투명 레이어(`rgba(0,9,5,0.55)`) → 텍스트 가독성 확보
- **텍스트 영역**: 중앙-좌측(영상의 빈 여백 자리)에 H1·통계·CTA 배치
- **성능**: mp4 720p, 파일 압축. 길면 로딩 지연 → 짧은 루프 유지
- 파일 배치: `public/hero-bg.mp4`, `public/hero-poster.jpg`

> ※ 영상·poster 파일은 힉스필드에서 다운로드해 `public/`에 직접 투입(사용자 작업).

## 5. 구현 우선순위 (Claude Code 작업 순서)

1. 셋업(Vite+React+TS+Tailwind) + **디자인 토큰 정의**(`tailwind.config` → B·PPT 공유)
2. Pretendard 폰트 셋업
3. 공통 레이아웃/내비/푸터 (B와 공유 전제로 분리)
4. 블록 1(Hero) — **배경 영상 + 모바일 fallback + 통계 카운트업**
5. 블록 2~5 순차
6. 반응형 점검(3 브레이크포인트)
7. 폼 외부 서비스 연결
8. `/mosaic-guard`, `/millog` 라우트는 빈 스텁 + 링크 연결
